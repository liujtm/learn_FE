# Vue 逐行带读

这份文档配合 `src/main.js` 和 `src/App.vue` 一起看。

如果你是后端开发，先用一句话记住这个项目：

Vue 会追踪你声明的响应式数据，并在它们变化时，自动更新模板里依赖这些数据的地方。

## 1. 从入口开始：`src/main.js`

```js
import { createApp } from 'vue'
import './style.css'
import App from './App.vue'
```

这几行的职责：

- `createApp`
  创建 Vue 应用实例
- `./style.css`
  全局样式
- `App.vue`
  当前页面的根组件

继续往下：

```js
createApp(App).mount('#app')
```

意思很直白：

1. 用 `App` 作为根组件创建一个 Vue 应用
2. 把它挂载到 `index.html` 的 `#app` 节点上

和 React 一样，Vue 也不是“自动接管整个网页”，而是接管一个挂载点里的内容。

## 2. 看主组件：`src/App.vue`

Vue 单文件组件通常分成几块：

- `<script setup>`
  写逻辑和状态
- `<template>`
  写视图结构
- `<style>`
  写样式

当前项目把样式放在外部 `style.css` 里了，所以 `App.vue` 里主要是前两块。

## 3. 先看 `<script setup>`

开头：

```js
import { computed, ref, watch } from 'vue'
```

这里导入了 3 个最值得先学的基础能力：

- `ref`
  声明响应式状态
- `computed`
  声明派生值
- `watch`
  监听变化并处理副作用

### 3.1 响应式状态：`ref`

```js
const name = ref('后端同学')
const requestCount = ref(0)
const latency = ref(120)
const logs = ref([])
```

`ref(...)` 的意思是：

- 把一个普通值包成响应式对象
- Vue 会跟踪谁在用它
- 当它变化时，依赖它的地方会更新

在脚本里读取时要写：

```js
requestCount.value
```

在模板里则直接写：

```vue
{{ requestCount }}
```

模板会自动帮你解包。

### 3.2 派生值：`computed`

```js
const totalCost = computed(() => requestCount.value * latency.value)
```

这行的重点是：

- `totalCost` 不是你手工维护的
- 它是由 `requestCount` 和 `latency` 推导出来的

所以只要依赖变化，它就会自动更新。

这个思路和后端里“根据源字段实时计算衍生字段”很像。

你不需要再手工写：

- 改了 `requestCount` 后顺手改一次 `totalCost`
- 改了 `latency` 后再补改一次 `totalCost`

`computed` 已经把这种同步工作接管掉了。

### 3.3 监听变化：`watch`

```js
watch([requestCount, latency], ([newCount, newLatency], [oldCount, oldLatency]) => {
  ...
})
```

这表示：

- 同时监听两个响应式来源
- 只要其中一个变化
- 回调就会执行

这里我们记录的是“变化前后值”：

```js
logs.value = [
  `requestCount ${previousCount} -> ${newCount}，latency ${previousLatency} -> ${newLatency}`,
  ...logs.value,
].slice(0, 5)
```

这段代码做了 3 件事：

1. 生成一条新日志
2. 插到数组最前面
3. 只保留最近 5 条

`watch` 很适合处理：

- 日志
- 请求
- 埋点
- 本地存储同步
- URL 参数同步

## 4. 再看 `<template>`

Vue 模板可以理解成“视图声明”。

它做的不是手工操作 DOM，而是告诉 Vue：

- 当前有哪些数据
- 它们应该显示在哪里
- 用户交互时要调用什么逻辑

### 4.1 输入框和 `v-model`

```vue
<input v-model="name" />
```

这个写法的效果是：

- 输入框显示 `name`
- 用户输入新值时，自动写回 `name`

可以把它理解成“读值 + 写值”的语法糖。

如果手工展开，思路有点像：

- `:value="name"`
- `@input="name = $event.target.value"`

### 4.2 `.number` 修饰符

```vue
<input v-model.number="requestCount" type="number" min="0" />
```

这里的 `.number` 作用是尽量把输入值转换成数字。

因为原始输入框的值默认是字符串。

如果没有它，你后面做乘法、加法时更容易遇到类型问题。

### 4.3 直接在模板里用派生值

```vue
<span>总耗时估算：{{ totalCost }} ms</span>
```

模板不用关心 `totalCost` 是怎么更新的。
它只负责“拿值显示”。

这也是 Vue 用起来顺手的地方：

- 状态变化
- 派生值变化
- 模板自动拿最新值

### 4.4 点击事件

```vue
<button type="button" @click="requestCount += 1">+1 request</button>
```

这是一个很典型的 Vue 事件入口。

你可以把它理解成：

1. 用户点击按钮
2. 修改 `requestCount`
3. Vue 发现依赖变化
4. `totalCost` 重新计算
5. 模板刷新
6. `watch` 被触发

### 4.5 列表渲染

```vue
<li v-for="item in logs" :key="item">{{ item }}</li>
```

这里是 Vue 最常见的列表渲染方式：

- `v-for`
  遍历数据
- `:key`
  给每一项稳定身份

和 React 一样，`key` 很重要，关系到列表更新时的识别逻辑。

## 5. 一次完整交互的运行链路

假设你把“请求数”从 `0` 改成 `3`：

1. 输入框收到用户输入
2. `v-model.number` 把值写回 `requestCount`
3. `requestCount` 这个 `ref` 发生变化
4. `computed(totalCost)` 自动重新计算
5. 模板中用到 `requestCount`、`totalCost` 的地方自动刷新
6. `watch([requestCount, latency])` 触发
7. 日志数组更新
8. 日志列表刷新

这里你能明显感觉到，Vue 更强调：

- 你声明数据关系
- 框架负责追踪和更新

## 6. 调试时建议怎么打日志

推荐看 3 个点：

### 6.1 输入事件附近

如果你后面把 `v-model` 拆开手写，就可以在事件里打日志。

### 6.2 `watch` 里

当前项目最适合在这里观察前后值变化：

```js
watch([requestCount, latency], ([newCount, newLatency], [oldCount, oldLatency]) => {
  console.log('watch fired', { newCount, newLatency, oldCount, oldLatency })
})
```

### 6.3 `computed` 依赖理解

虽然一般不建议在 `computed` 里做副作用，但你可以临时打印帮助理解：

```js
const totalCost = computed(() => {
  console.log('recompute totalCost')
  return requestCount.value * latency.value
})
```

这样能感受到“依赖变了才重新算”的味道。

## 7. Vue 和 React 在这个例子里的感受差异

你作为后端同学，初学时可以先这样粗略分辨：

### React 更像

- 组件函数反复执行
- 每次重新拿状态算 UI

### Vue 更像

- 先声明响应式数据之间的依赖
- 框架跟踪这些依赖并更新受影响部分

这两个说法都不是完整定义，但足够帮助你建立第一层直觉。

## 8. 读完后建议你立刻做的练习

1. 新增一个 `errorRate` 输入框
2. 增加一个 `computed`：`errorCount`
3. 增加一个 `watch`：当 `latency > 500` 时写一条告警日志
4. 把日志列表提取成独立组件
5. 再写一个 `watchEffect`，体会它和 `watch` 的区别

做完这些之后，Vue 的响应式模型会变得非常具体。
